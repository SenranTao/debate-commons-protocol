# 辩论信息公地协议（DCP）章程

> Debate Commons Protocol — Constitution
>
> v0.1 · 待第一批签署者共同审议生效
>
> **用语规范**：本章程正文使用中文。协议专用术语——Badge、Probation、Terminated、Banished、Active、Copyleft、Maintainer、Contributor——在正文中首次出现时以中文释义标注，此后直接使用英文原文。技术术语（LLM-Wiki、Git、PR、Fork、Base36）不翻译，专有名称（CC BY-SA 4.0、Creative Commons）不翻译。

## 序言

我们——本协议的签署者——观察到以下事实：

国内大学英语辩论的参与人口持续扩大，学习资源的绝对数量持续增长，但辩手水平的提升未能与之同步。这一悖论的结构性根源之一，是**信息的私有化**：每一个社团、每一个赛事、每一个线上社群，都作为独立的信息容器运行，容器之间不存在系统性的互通机制。

我们进一步观察到：呼吁分享的道德劝说未能奏效——因为信息私有化对信息持有者有利。在缺少规则约束的条件下，分享是个体损失，不分享是理性选择。

我们不认为这是任何个体的道德失败。我们认为这是**规则设计的缺失**。

本协议的设计目标，是创建一个**去中心化的资源共享网络**，通过条件性互惠的规则设计，使得信息共享不再是损失——信息不共享才是。本协议本身不作为组织存在，没有中央权威，没有管理层级。它是一个标准：遵守者获得网络的权益，违反者失去网络的权益。

本协议的哲学渊源，承认 Richard Stallman 的自由软件运动（Copyleft）、Elinor Ostrom 的公共资源自治理论、以及 B-Corporation 的公开认证与信任信号机制。

## 第一条：名称、定义与适用范围

### 1.1 名称

本协议全称为"**辩论信息公地协议**"，英文名 *Debate Commons Protocol*，缩写 **DCP**。

### 1.2 核心定义

**协议**：本文件所载的全部规则与标准。协议本身不是一个法律实体、不是一个组织、不是一个社团。它是一个公开标准——任何满足条件的个人或团体，可以自愿订阅并遵守。

**成员**：自愿订阅本协议并履行其义务的个人或团体。成员资格是一种**状态**，不是一种身份——成员不隶属于"DCP组织"，因为不存在DCP组织。

**LLM-Wiki**：协议的公共知识库。以结构化 Markdown 文件的形式托管在公开 Git 仓库中，支持人类阅读和 AI 代理读写。LLM-Wiki 是协议的**唯一真相来源**：成员目录、协议文本、共享资源索引、贡献记录——均以 Wiki 页面形式存在，不在任何其他位置。

**Recognition Badge**：成员在订阅协议后获得的公开认证标识。Badge 表明该成员当前的合规状态。任何第三方可通过访问 LLM-Wiki 公开目录验证 Badge 的真实性。

**共享 Database**：由所有成员共同建设和维护的辩论资源池，包括但不限于：ballot 反馈数据库、motion bank、训练方法论文档、裁判分析报告、比赛 transcript、AI coach 使用方法与 prompt 分享。共享 Database 的全部内容以 CC BY-SA 4.0 许可发布（见第3.3款）。

**Preferential Access**：成员之间互相提供的优惠权益，包括但不限于：赛事报名费折扣、优先参赛名额、AI coach 服务优先访问。

**CC BY-SA 4.0**：Creative Commons 署名-相同方式共享 4.0 国际许可协议。本协议所有内容资源的法定许可框架（见第3.3款）。

### 1.3 适用范围

本协议适用于所有自愿订阅的中国大学英语辩论社群成员——包括个人辩手、辩论社团、赛事主办方、线上 spar 群运营者。协议的范围仅限于成员在辩论社群活动中的行为，不介入成员的私人生活、非辩论相关的商业活动、或成员在协议成立之前的既有约定。

## 第二条：成员资格

### 2.1 订阅条件

任何在辩论社群中具有可验证身份的个人或团体，均可申请订阅本协议。申请方式为通过协议指定的接入渠道提交以下信息：

1. 名称（由申请人自行设定，不要求实名）
2. 身份类型：Individual（个人）/ Community（团体）
3. 可验证的联系信息：Individual 提供个人联系方式（微信号等）；Community 提供所运营社群的公开信息（微信群、公众号、飞书群等）
4. 入会贡献（选择其一）：
   - 提交一份资源——ballot 反馈、motion bank、训练方法文档、裁判分析等任何对辩论学习有价值的材料；或
   - 完成一份匿名痛点问卷——描述申请人在英辩学习过程中的痛点、困惑和需求（问卷由协议统一设计，答案不关联成员身份）。如果你不确定能贡献什么，问卷是最低门槛的入会贡献。
5. 同意本章程的声明

### 2.2 不设准入门槛

申请不设评审门槛。申请信息齐全、声明真实 → 批准。拒绝申请的唯一理由是信息虚假或不全，或使用的名称无法被验证为申请人在辩论社群中的既有身份。

### 2.3 入会渠道

入会申请的具体渠道、技术实现与流程，见运营手册 `[[protocol/operations]]`。

### 2.4 成员类型

| 类型 | 说明 | 核心义务 |
|------|------|----------|
| **Individual** | 以个人身份加入 | 个人义务：资源贡献（不运营群组的 Individual 不承担月度资讯转发义务） |
| **Community** | 社团、赛事组织、spar 群等团体 | 团体义务：群内信息互通（置顶）+ 资源贡献 + 赛事互助 + 月度资讯转发。由团体负责人或其指定成员履行 |

一个 Community 履行的义务取决于其实际行为——如果一个 Community 办了比赛，它自然触发赛事互通义务（第四条第4.3款）。义务不是由类型定义的，是由行为触发的。

### 2.5 成员 ID

每位成员在加入时被分配一个唯一 ID，格式为 `DCP-XXXX`，其中 XXXX 为四位 Base36 编码（数字 0-9、字母 A-Z）。ID 由成员的内部顺序号经质数乘数与固定偏移混淆后编码生成——例如第 1 位成员的 ID 为 DCP-0GGS，第 2 位为 DCP-0MKH。人眼无法判断任何两个 ID 之间的加入顺序，但协议系统可反向解码还原内部顺序号。

ID 不可更改，不因成员退出而回收。ID 本身不携带加入日期、身份类型或其他个人信息。它是成员在 LLM-Wiki 中的唯一标识符，Badge 的验证依赖于 ID。

### 2.6 身份验证

成员名称由申请人自行设定，无须为法定姓名。但申请人使用的名称必须是其在辩论社群中被广泛使用的、可追溯的身份。若发现申请人使用的身份无法被验证为申请人自身，申请无效。冒用他人身份 → 诚信义务违反 → 立即 Banished（见第七条第7.2款）。

### 2.7 Community 与 Individual 的关系

Community 成员资格与 Individual 成员资格是**平行的、独立的**两条轨道。两者互不隶属、互不继承、互不影响。

一个 Community 的社员，不因该 Community 加入协议而自动成为 DCP 成员。他们通过所属 Community 的置顶帖访问共享资源、享受网络福利——但他们没有自己的 Badge、没有自己的贡献分、没有投票权。

Community 中的个人可以另行以 Individual 身份单独加入协议。此时该个人同时持有两个成员资格——两个资格独立计算贡献分、独立承担义务、独立享有权利。Community 退出不影响其 Individual 成员资格；Individual 退出不影响其所属 Community 的成员资格。

Community 的贡献与 Individual 的贡献分开记录。Community 负责人代 Community 提交的资源归属于 Community 的贡献分。个人以自己的 Individual 身份提交的资源归属于个人的贡献分。

## 第三条：LLM-Wiki

### 3.1 什么是 LLM-Wiki

LLM-Wiki 是协议的**基础设施**——一个以结构化 Markdown 文件组成的公开知识库，托管在 Git 仓库中。它既面向人类浏览，也面向 AI 代理读写。人类可以通过网页浏览，AI 可以通过结构化字段进行索引和推理。

LLM-Wiki 是协议的**唯一真相来源**。成员目录、协议文本、共享资源索引、贡献记录——一切以 Wiki 页面形式存在于此，不在任何其他位置。不存在"群里说了一个版本、Wiki 上是另一个版本"的情况——Wiki 上的版本就是唯一版本。

LLM-Wiki 的目录结构如下：

```
wiki/
  members/         — 成员目录（每位成员一个页面，含 Badge 状态）
  protocol/        — 章程正文、运营手册、规则修改提案
  resources/       — 共享 Database 资源索引
  events/          — 赛事与活动记录
  concepts/        — 概念与理论基础
```

所有页面以 YAML frontmatter 标注元数据、以 [[wikilinks]] 实现内部链接。这使 LLM-Wiki 既可被人类按目录浏览，也可被 AI 一键加载全部上下文进行推理。

### 3.2 公开性

LLM-Wiki 的**全部内容对全世界公开**。任何人——无论是否为成员——均可阅读。成员目录、资源索引、贡献记录——全部可被查证。公开性不是策略，是协议的存在理由：信息不对称的消除，以信息对称为基础。

LLM-Wiki 的写入权限、技术栈、月度资讯等运营细节，见运营手册 `[[protocol/operations]]`。

### 3.3 内容许可：CC BY-SA 4.0

LLM-Wiki 的全部内容——包括共享 Database 中的资源、成员贡献的方法论文档、ballot 反馈数据库、motion bank、训练材料——均以 **Creative Commons 署名-相同方式共享 4.0 国际许可协议（CC BY-SA 4.0）** 授权发布。

CC BY-SA 4.0 的含义：

- **署名（BY）**：任何使用、修改或分发本库内容的主体——**无论是个人还是机构、无论是否出于商业目的**——都必须明确注明原作者，并附上协议信息主页的链接，使第三方可查证内容来源与许可条款。
- **相同方式共享（SA）**：任何基于本库内容创作的衍生作品，必须以相同的 CC BY-SA 4.0 许可证发布。此即 Copyleft 的"传染"机制在内容领域的具体实现。

这意味着：一名辩手将自己的训练笔记发给朋友——需要注明"本文内容来自 DCP 共享数据库，协议信息见 [链接]"。一家培训机构将 ballot 反馈用于付费课程——需要注明来源、在平台上公开其改版内容、以 CC BY-SA 4.0 许可其衍生作品。署名义务不因使用规模或是否收费而改变。

**访问即同意**：共享 Database 的访问入口设置显式许可声明，明确告知使用者——访问即表示同意 CC BY-SA 4.0 条款。若使用者不同意条款，其无权访问本库内容。

任何主体违反 CC BY-SA 4.0 许可条款——包括未署名、未公开衍生作品、将衍生作品以非 CC BY-SA 许可发布——均构成著作权侵权。违规记录将公开发布在 LLM-Wiki 中，永久留存。协议的强制力首选社会声誉机制——CC BY-SA 为法律维权提供文本基础，但诉讼不构成协议的常规操作。

**版权归属与协调执行**：每份资源的版权属于其贡献者个人。协议不要求版权转让。贡献者在提交资源时，即授权协议维护者代表其发送侵权告知函、并在 LLM-Wiki 中发布公开违规记录。协议维护者不获得代表贡献者提起诉讼的权利。

**非成员内容的边界**：CC BY-SA 4.0 仅适用于协议成员自己创作的内容。如果一名成员收录了一份非协议成员公开发布的资源——例如一份网络上流传的 debate lecture spreadsheet——该资源的原始版权仍属于原作者，协议不对其主张 CC BY-SA 许可。成员的贡献在于**组织、索引、注释**这些增值部分——而非将原作者的内容据为己有。成员在贡献此类资源时，必须在页面上明确标注资源来源、原作者信息，并区分"原创贡献"与"索引引用"。AI 在质量初评时将自动检测直接复制他人内容的投稿并将其标记为需要人工审核。

**成员责任声明**：每位成员对其上传至共享 Database 的内容负全部责任。协议不对成员上传内容的版权合规性、准确性或合法性承担担保义务。如果一名成员上传了其无权授权的内容——例如未经许可转载他人的版权材料——该成员自行承担由此产生的法律后果。协议维护方在收到有效的版权侵权通知后，将从共享 Database 中移除相关内容，并依据违规处理规则（运营手册 `[[protocol/operations]]`）对上传者进行处理。

## 第四条：成员义务

### 4.1 信息互通义务（基石条款）

任何运营辩论相关群组（微信群 / QQ群 / Discord / 飞书群 / 其他）的成员，必须在自己运营的每个群内**长期置顶**一份"协议资源中心"文档。该文档为协议统一维护的公开页面，包含：

- 所有协议成员的群组入口
- 所有协议成员赛事的最新信息
- 共享 Database 的访问链接
- 协议公开目录的链接（供查证 Badge）

置顶义务是本协议的**基石条款**。未履行置顶义务 → 直接进入 Probation（考察期，Badge 状态降级）。它是 Copyleft 的"传染"机制在社群层面的具体实现：你享受了网络的互通福利，你的群就必须成为网络的一个信息节点。

### 4.2 资源回馈义务

每位成员每12个月必须向共享 Database 贡献**至少一份**资源。资源类型不限（ballot反馈、motion bank、训练方法、裁判分析、transcript整理、AI工具使用方法等），质量以协议规定的最低标准为通过线。一份也算达成义务。

连续12个月未贡献 → Probation → 再宽限期内仍未贡献 → Terminated（终止资格，Badge 失效，但可重新申请入会）。

不设贡献量上限。这是"至少一杯水"的逻辑，不是"要付入场费"。

### 4.3 赛事互通义务

赛事主办方成员，必须在赛事官方宣传材料中注明协议成员资格，并为所有其他协议成员提供统一的 Preferential 报名费率。费率的折扣幅度由主办方自行决定——不存在强制折扣比例。义务在于"提供费率"，不在于"提供多少"。

### 4.4 月度资讯传播义务

协议每月向成员推送一份月度资讯简报——包含本月赛事日历、新入库高质量资源、新加入成员。

**任何运营群组的成员**必须在自己的群组和主要传播渠道中**转发该简报**。不运营群组的 Individual 成员不承担此项义务。

转发义务存在——但不是"你自己写"，而是转发协议统一生成的简报。未转发月度资讯 → 当季度贡献评分的传播维度归零。连续两个季度未转发 → Probation。这是一项强制执行义务——传播不是情分，是你在网络里的结构性功能。

### 4.5 诚信义务

任何成员不得在非协议群组中冒用协议 Badge、不得伪造贡献记录、不得将自己的资源标注为"协议共享资源"但未纳入共享 Database。

## 第五条：成员权利

### 5.1 访问权

所有成员可访问共享 Database 的全部内容。

### 5.2 Preferential Access

赛事主办方成员必须为所有协议成员提供 Preferential 报名费率。费率折扣幅度由主办方自行决定。不提供 → 违反赛事互通义务（第四条第4.3款）。

享受 Preferential Access 的前提：成员至少有一份被 approve 的资源贡献。未在12个月内完成首次贡献的成员，仍可访问共享 Database，但不可享受 Preferential Access。贡献是门票——不是入会的门票，是优惠的门票。

### 5.3 信息触达权

所有成员有权将自己的赛事/活动/资源信息发布到协议中央日历，并通过置顶资源帖触达所有其他成员的群组。

### 5.4 投票权

成员基于其贡献累计获得的功能权限，享有对应的投票权：

- **终止投票**：Contributor 可参与针对个人的投诉投票——决定是否终止或除名某成员。这是社区自我管理的核心机制：成员的去留不由单独个人决定，而由贡献者群体共识裁决。
- **否决权**：Founding Maintainer（见第九条第9.2款）对协议规则修改拥有否决权。

投票的具体流程与阈值见运营手册 `[[protocol/operations]]`。规则修改在创始维护者 active 期间不设投票机制（见第九条第9.2款）。

### 5.5 退出权

任何成员可随时退出协议，无需说明理由。退出 → 从公开目录移除 → Badge作废 → 不再承担任何义务 → 不再享有任何权利。退出前的贡献永久保留在 LLM-Wiki 中。退出不是耻辱——这是自由订阅的必然结果。

## 第七条：Recognition Badge

### 7.1 Badge 的根本属性

Badge 是一个**可验证的公开信号**。它不授予任何法律权利——它是声誉基础设施。任何第三方可通过访问 LLM-Wiki 公开目录，通过搜索成员 ID 验证 Badge 的真伪与当前状态。

### 7.2 Badge 状态

| 状态 | 含义 | 显示 |
|------|------|------|
| **Active** | 良好信誉，履行义务 | 绿色标识 |
| **Probation** | 信任受损，部分权益暂停。可在规定期限内纠正 | 黄色标识 + 原因简述 |
| **Terminated** | 协议终止，成员资格丧失。可重新申请（等待期见第八条） | 灰色标识 + 终止日期 |
| **Banished** | 永久除名。不可重新申请。不可被其他成员附议恢复 | 黑色标识 + 除名日期 + 原因简述 |

Banished 是协议的终极制裁。适用于：冒用Badge、伪造贡献记录、蓄意破坏LLM-Wiki内容、严重损害协议声誉或成员信任的恶意行为。Banished 一经判定，不可上诉、不可逆转。

Terminated 是"你暂时不属于这个网络了，以后可以回来"。Banished 是"你的名字将永远以黑色出现在公开目录上"。

### 7.3 功能权限

成员基于贡献分获得对应的功能权限（**Member** / **Contributor** / **Maintainer**）。功能权限的具体分值与对应权益见运营手册 `[[protocol/operations]]`。

功能权限不是社会身份、不是头衔。"我是 Contributor"不是身份声明——是"我为共享池做了贡献，因此我的投稿走快速通道、我可以审核他人投稿"的功能描述。任何成员不得以其功能权限论证其意见权重（见第9.5款）。

## 第八条：监督与合规

### 8.1 社区举报

协议不设中央监督机构、不设定期自检、不设随机抽查。合规监督的唯一机制是**社区举报**。

任何成员可通过接入渠道举报其他成员的违规行为。举报匿名提交、人工核实。核实确认 → 按违规处理规则执行。无效举报累计三次 → 举报者贡献分扣减（防止滥用举报）。

### 8.2 违规处理

违规的处理程序、Probation 期限、Terminated 等待期、Banished 的判定流程及争议解决机制，见运营手册 `[[protocol/operations]]`。

## 第九条：治理

### 9.1 无中央权威

协议没有主席、没有理事会、没有组织架构。协议是一个**标准**——规则本身具有最高权威。协议的"管理者"仅为 LLM-Wiki 的技术维护者，不对协议规则拥有解释权或裁定权。

### 9.2 规则修改

协议的规则修改由**创始维护者**（Founding Maintainer）最终决定。流程如下：

**提案**：任何 Maintainer 或 Contributor 可在 LLM-Wiki 中提交规则修改提案（`wiki/protocol/proposals/`）。提案须说明修改内容及理由。创始维护者也可以自行发起修改，无需他人提案。

**审核**：创始维护者审阅提案，决定批准或驳回。批准 → 修改生效，LLM-Wiki 同步更新。驳回 → 提案归档，驳回理由公开发布。

**公开理由**：创始维护者的**每一项**规则修改决定——无论是批准提案还是自行发起——都必须附有公开的修改理由。不发布理由的修改不具有效力。公开理由是创始维护者对社区的唯一义务：不是善良，是**可被审视**。

DCP-0GGS（森然）为本协议的创始维护者。创始维护者身份可以自愿转让或放弃。转让须指定一名现任 Maintainer 作为继任者。不可通过投票罢免创始维护者。

创始维护者的存在与第9.1款"无中央权威"原则之间存在**创造性张力**——这是协议的刻意设计，而非矛盾。协议的权威属于规则本身，而非任何个人。但规则不是凭空产生的——它有人设计、有人维护、有人在最坏情况下承担最大的损失。创始维护者对规则修改的控制是对这一责任的承认。Contributor 和 Maintainer 的提案权则确保规则不是独角戏——社区的声音可以被听到，即使最终决定权在一人之手。

**治理演进**：当协议达到一定规模且社区治理条件成熟时，创始维护者可主动引入 Maintainer 投票机制参与规则修改，或将规则修改权移交至 Maintainer 集体。这一演进由创始维护者判断和启动——不设自动触发条件。

### 9.3 协议分叉

任何成员如不同意创始维护者的规则修改，有权 Fork LLM-Wiki 仓库，在 Fork 版本上实施自己的修改，并以新的协议名称独立运行。Fork 后，原协议与新协议互不隶属，互无权利与义务。

创始维护者的规则修改权**不受分叉权约束**——不赞同 → 分叉，而非强制原协议接受不同的方向。这是去中心化的终极保障。

Fork 的权利是 Copyleft 哲学的具体体现：不服从 → 另起炉灶 → 社区看谁的规则更优。退出不是分裂——是**规则的自然选择**。如果创始维护者持续做出糟糕的决策，社区会用分叉投票。如果社区找不到足够的人愿意分叉——那决策可能并非糟糕。

### 9.4 非权威原则

任何成员不得以其功能权限（Contributor、Maintainer、Founding Maintainer）论证其意见的权重。在协议规则的讨论、投票和争议解决中，所有参与者的意见权重相等——不论其贡献分或功能权限。等级的差异仅限于功能权限（审核、写入、终止投票），不延伸至治理话语权。

本条款不依赖外部强制力——它的存在本身构成协议的伦理锚点。任何成员引用自己的等级来压制他人意见时，其他成员有权援引本条款。

## 第十条：退出与终止

### 10.1 主动退出

成员可随时申请退出。退出不可撤回（可以重新申请，但重新申请 = 重新从 Member 级起步，原贡献分归零）。

### 10.2 被动终止与除名

成员因违反章程第八条而被 Terminated。Terminated 后，成员可重新申请入会——前提是等待期满（常规违规：3个月；严重违规：12个月）且获得至少三名现任成员的附议。

成员因违反章程第八条而被 Banished。Banished 不可重新申请、不可上诉、不可逆转。

### 10.3 退出与除名后的数据

成员退出的信息记录（名称、ID、贡献页）**永久保留**在 LLM-Wiki 中，但 Badge 状态标记为 Terminated 或 Banished。贡献内容永久保留且继续被协议成员使用。退出成员不再维护自己贡献的资源——相关页面由 AI 自动标记为"未维护"。

## 第十一条：生效与修订

### 11.1 初版生效

本章程 v0.1 在获得**至少五名**签署者的一致同意后生效。生效后的首次修订窗口为生效日后的第90天——前90天内不修改章程，给予社区适应期。

### 11.2 此后修订

生效后，章程按第九条规则修改。每次修订形成新版本号（v0.2, v0.3 ... v1.0）。LLM-Wiki 保留所有历史版本，可查可溯。

### 11.3 不设 Sunset Clause

本章程不设自动失效日期。协议的存废由其成员决定——当最后一名成员退出时，协议自动终止。LLM-Wiki 作为历史档案永久保留。
